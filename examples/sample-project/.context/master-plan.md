# Sample Project 总体计划

## 目标

- 做一个可持续推进的小型产品原型，并在实施阶段始终维持清晰的项目上下文。

## 非目标 / 明确不做

- 不构建完整项目管理平台。
- 不引入数据库、服务端或第三方依赖。
- 不在设计门禁未通过时进入功能实现。

## 阶段与里程碑

### Phase 1

- 初始化 `.context/`，明确愿景、设计、状态、任务拆解和风险。
- 建立第一轮工作日志与问题复盘样例。

### Phase 2

- 进入真实功能开发，保持日志、Bug 与决策同步更新。

## 验收标准

- 项目进入实施前，所有核心 `.context` 文件齐全且 `VISION + DESIGN` 通过门禁。
- 开发过程中，每个明确步骤至少有一条工作日志。
- 工程异常具备根因与预防记录。
