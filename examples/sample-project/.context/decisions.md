# 决策记录

> 记录高影响决策，不要只记录结论，要写背景、取舍和后续约束。

## ADR-0001：使用 `.context/` 作为工程中枢

- 状态：accepted
- 日期：2026-04-07
- 背景（Context）：项目会跨多个会话推进，需要稳定上下文。
- 决策（Decision）：计划、任务推进、Bug 复盘、决策记录全部统一进入 `.context/`。
- 取舍（利弊，放弃了什么）：获得连续性和可验证纪律；放弃完全自由的即兴实现。
- 后果（Consequences）：进入真实实施前必须先读 `current-status.md`、当前活跃日志分片和 `design.md`。
- 取代关系（Supersedes）：无
